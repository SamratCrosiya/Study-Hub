What are CSS Variables?
CSS Variables let you store values (like colors) and reuse them everywhere. This makes your code cleaner and easier to update.

üìù Properly Formatted Code:

/* Define your color palette */
:root {
    /* Primary Colors */
    --primary: #2563eb;
    --primary-light: #3b82f6;
    --secondary: #8b5cf6;
    --accent: #f59e0b;
    
    /* Text Colors */
    --text-white: #ffffff;
    --text-gray: #94a3b8;
}

/* Using the variables */
.button {
    background: var(--primary);
    color: var(--text-white);
    padding: 15px 30px;
    border-radius: 50px;
}

.button:hover {
    background: var(--primary-light);
}

üîç Line-by-Line Explanation:

Line 1-2: We define our root variables inside :root
Line 4-7: Each color gets a descriptive name like --primary
Line 13: We use the variable with var(--primary)
Line 19: On hover, we change to the lighter version
üí° Why This is Better: Instead of typing #2563eb 50 times, you define it once and use var(--primary). If you want to change the color later, you only update one line!

3Ô∏è‚É£ CSS Animations - Make Things Move!

What are Keyframe Animations?
Animations let you move, rotate, scale, or change elements over time. They make your website feel alive!

üìù Step 1: Define the Animation:

/* Define what the animation does */
@keyframes fadeInUp {
    /* Starting state (0% or "from") */
    from {
        opacity: 0;                    /* Invisible */
        transform: translateY(50px);   /* 50px below */
    }
    
    /* Ending state (100% or "to") */
    to {
        opacity: 1;                    /* Fully visible */
        transform: translateY(0);      /* Original position */
    }
}

üìù Step 2: Apply the Animation:

.my-box {
    /* Apply the animation */
    animation: fadeInUp 1s ease-out;
    
    /* Breaking down "fadeInUp 1s ease-out":
       - fadeInUp    = name of animation
       - 1s          = duration (1 second)
       - ease-out    = timing (slow at end)
    */
}

‚ö° How It Works:

0% (from): Element is invisible (opacity: 0) and 50px below
25%: Element is 25% visible, 37.5px below
50%: Element is 50% visible, 25px below
75%: Element is 75% visible, 12.5px below
100% (to): Element fully visible, at original position
üí° Common Timing Functions:
‚Ä¢ ease - Slow start, fast middle, slow end
‚Ä¢ ease-in - Slow start
‚Ä¢ ease-out - Slow end
‚Ä¢ ease-in-out - Slow start and end
‚Ä¢ linear - Constant speed

4Ô∏è‚É£ JavaScript Functions - Make Things Interactive

What are Functions?
Functions are reusable blocks of code that perform specific tasks. They make your website interactive!

üìù Simple Function Example:

// Define a function
function showMessage() {
    // Get the element with id="greeting"
    const element = document.getElementById('greeting');
    
    // Change the text inside it
    element.innerHTML = 'Hello, World!';
    
    // Change the color
    element.style.color = '#667eea';
}

// Call (use) the function
showMessage();

üìù Function with Parameters:

// Function that accepts input
function greetUser(name) {
    // 'name' is a parameter - we can pass any name
    const message = 'Hello, ' + name + '!';
    
    // Show alert with personalized message
    alert(message);
}

// Call with different names
greetUser('Aagam');    // Shows: "Hello, Aagam!"
greetUser('Student'); // Shows: "Hello, Student!"

5Ô∏è‚É£ API Integration - Connect to External Services

What is an API?
API (Application Programming Interface) lets your website talk to other services. Like asking Google's AI to answer questions!

üìù Making an API Request:

// async means this function can "wait" for things
async function sendMessage() {
    // 1. Get user's message from input field
    const message = document
        .getElementById('chatInput')
        .value;
    
    // 2. Get API key
    const apiKey = document
        .getElementById('apiKey')
        .value;
    
    // 3. Prepare the data to send
    const requestBody = {
        contents: [{
            parts: [{
                text: message
            }]
        }]
    };
    
    // 4. Build the API URL
    const apiUrl = 
        'https://generativelanguage.googleapis.com/' +
        'v1beta/models/gemini-pro:generateContent' +
        '?key=' + apiKey;
    
    try {
        // 5. Send request to API (wait for response)
        const response = await fetch(apiUrl, {
            method: 'POST',
            headers: {
                'Content-Type': 'application/json'
            },
            body: JSON.stringify(requestBody)
        });
        
        // 6. Convert response to JSON
        const data = await response.json();
        
        // 7. Get AI's answer from response
        const aiAnswer = data
            .candidates[0]
            .content
            .parts[0]
            .text;
        
        // 8. Show AI's answer on screen
        displayMessage(aiAnswer);
        
    } catch (error) {
        // 9. If something goes wrong, show error
        console.error('Error:', error);
        alert('Failed to connect to AI');
    }
}

üîç What Each Part Does:

Steps 1-2: Get the message and API key from input fields
Step 3: Package the message in the format the API expects
Step 4: Build the complete API URL with your key
Step 5: Send the request and wait for response
Step 6-7: Extract the AI's answer from the response
Step 8: Display the answer to the user
Step 9: Handle errors if something fails
üí° Key Concepts:
‚Ä¢ async/await - Wait for operations to complete
‚Ä¢ fetch() - Send HTTP requests
‚Ä¢ try/catch - Handle errors gracefully
‚Ä¢ JSON - Data format for API communication
‚Ä¢ POST - Method for sending data
‚Ä¢ headers - Additional request information
